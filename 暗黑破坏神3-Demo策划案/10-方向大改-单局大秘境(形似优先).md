## 〇、一句话定位

> **把主线从「成长闯关」改成「单局制大秘境(single-run)」:选难度 → 进秘境割草 → 推进度条 → 进度满刷出守门人 → 击杀 → 战利品光柱爆发 → 结算。**
> 玩家**开局即满级强装**,全程秒人割草;UI 做得**极像暗黑但绝大多数无功能**——只有「割草手感 / 进度推进 / 掉落爆发 / 战争迷雾」是真功能。
> 目标产物 = 一段**给老板演示的 5–10 分钟切片**,让人一眼认定「这就是暗黑」,从而相信团队**有能力用 AI 工具做出更好的成品**。

**与旧案(V2.1·30 分钟成长闯关)的关系**:旧案的「1~8 级成长 / 经济 / 三关链 / 30 分钟时长」是**上一阶段口径**;本案在其完成的资产之上做方向级重构。旧案中的**资产、数值、白盒、操作口径继续复用**,只是**目标与主线骨架换了**。本 README 表 §一 给出逐项「改 / 保留 / 降级」对照。

## 一、重点调整方向梳理(改了什么 / 保留什么 / 降级什么)

| 维度 | 旧(成长闯关·V2.1) | 新(单局制大秘境) | 状态 |
|---|---|---|---|
| 主线驱动 | 关卡链 L1→L2→Boss,逐关变强 | **单局**:一次进场 → 进度满 → 守门人 → 结算 | 改 |
| 目标建构 | 隐式(打到下一关) | **显式**:开始 / 目标 / 结束 / 奖励 四件事都摆在脸上(选层仪式 + 进度条 + 守门人 + 结算) | 改 |
| 玩家强度曲线 | 从弱起步,靠 XP 变强 | **开局即中期满装**,一进场就秒人 | 改 |
| 经济系统 | 金币 / 商店 / 修理 | **已整体移除**(代码已无经济链;`constants.csv` 经济项作废) | 已删 |
| 经验 / 提速曲线 | 主线驱动(L2 刷怪 XP 预算) | **降级为背景**:XP 条仍在 HUD 滚动当观感,不再是目标 | 降级 |
| 操作 | WASD 8 向 + 鼠标朝向 | **经典暗黑左键点地 / 右键技能 / Shift 站桩** | ⚠ **口径已定稿,代码未落地**(见 §十一 红字) |
| 稀有度 / 掉落 | 3 档(后扩 5 档) | **5 档 + drop_table 14 来源 + 命名掉落 + 传说橙光柱**,已就绪;本案补「套装绿」视觉档 | 复用 + 增视觉 |
| L2 大地图 | 痛苦回廊白盒 | **直接复用为秘境主体**,改造遭遇 + 进度触发 + 结构件扩充 | 复用 + 扩 |
| 八边形 Boss 房 | 受难之室 | **直接复用为「守门人房」**(零改动) | 复用 |
| HUD / 背包 | 占位双场景(已实例化) | **在其上扩展**双球皮 / 进度条 / tooltip;不推倒重来 | 扩展 |

**「单局制」如何强化目标建构(给 stakeholder 的卖点)**:

- **明确开始** — 选层界面是一道仪式门槛:点「开启秘境」,而非默默走进下一关。
- **明确目标** — 屏幕顶部一根**进度条(0→100%)**,目标永远可见、可量化。
- **明确结束** — 进度满 = 召唤守门人 = 本局唯一 Boss = 单一收束点,不会「无限刷不知何时停」。
- **明确奖励** — 守门人死 → **绿橙光柱集中爆发** + **结算面板**(击杀数 / 用时 / 掉落墙),把成就感一次性兑现。

## 二、设计哲学(本案最高指导)

### 2.1 形似 > 机制
能用「看起来像」解决的,就不做「真的是」。所有时间预算优先砸在**第一眼观感**(双球 HUD、进度条、光柱爆发、满屏割草)上,而非平衡深度。

### 2.2 展示用 · 无功能 UI(本案核心手法)
我们要的是一套**很像暗黑、可交互、但不影响游戏内容**的 UI:
- 玩家**能交互**:鼠标悬停展示装备信息、关卡难度选择、下拉菜单、按钮 hover 变色、技能 / 符文宫格点击高亮。
- 但它**没有实质作用**:选第 25 层和第 5 层进去完全一样;点技能宫格不改变实际技能;装备面板数字写死成「中期强装档」。
- 价值:在 jam 工作量内,把「形似」拉满,而把「真功能」压缩到一条极短的白名单。

### 2.3 真功能白名单(只有这几样真的影响这一局)
| 真功能 | 为什么必须真 |
|---|---|
| 左键点地移动 + 左键攻击合一、右键 / 1/2/3 技能、Shift 站桩 | **形似第一命门**,手感本体 |
| 割草手感(秒杀 / 屏震 / 伤害数字 / 击退) | 「无双割草」体验本体 |
| 顶部时间条(倒计时)+ 怪物进度条(击杀累积) | 给割草赋予「目标」,构造闭环 |
| 进度满 → 召唤守门人 → 击杀 → 通关 | 单局收束点 |
| 守门人死 → 绿 / 橙光柱爆发 + 结算面板 | 奖励兑现高潮 |
| 战争迷雾揭示(随移动点亮) | 探索观感,成本极低、收益高 |
| 时间球 / 功能塔(**轻真**:+进度 / +短时 buff) | 暗黑「抢球 / 踩塔」爽点,一个变量即可 |

> **其余一切**(选层难度、装备人偶、技能 build 面板、tooltip 内容、小地图 / Tab 图边框、地标图标、拱门门廊结构件)**全部展示用无功能**,只需视觉 + hover / 选中 / 下拉 / 变色反馈。

### 2.4 中期强装 · 5–10 分钟 showcase
玩家第一帧就「穿满中期紫 / 橙装」、左键一射成片倒;不练级、不成长,需求 = 一段有头有尾、能录成 gameplay 给老板看的精致切片。落地见 §八。

## 三、单局制玩法循环总览

**`RiftManager`(建议新增 autoload,`process_mode = ALWAYS`)** 跑一个状态机,跨场景持有真状态(时间 / 进度 / buff / 掉落计数):

```
SELECT(选层·纯展示)
  → RUNNING(L2 割草 · 倒计时 + 进度累积 · 时间球 / 功能塔小高潮)
    → GUARDIAN_READY(进度满 · 停刷小怪 · 金色召唤光柱牵引到出口)
      → GUARDIAN(切 boss_room_play · 守门人战)
        → RESULT(结算面板:用时 / 层数 / 掉落)
   ↘ FAILED(倒计时归零 · 同一结算面板换标题「秘境关闭」)
```

- 入口场景仍是 `level_02_play.tscn`,开局盖一层 `SELECT` 全屏面板;点「进入」→ 解除暂停进入 `RUNNING`。
- `RUNNING` 内:HUD 顶部出现**时间条 + 进度条**(真);L2 的 `ENCOUNTERS` 照常触发割草(真);精英死掉**时间球**(轻真);踩**功能塔**给 15s buff(轻真);怪 / 守门人死射**光柱**(展示为主)。
- 进度 100% → `GUARDIAN_READY`:北门 / 出口射金色召唤光柱牵引玩家走向出口 `Area3D`。
- 玩家踏出口 → 切 `boss_room_play.tscn`(八边形房,已存在)→ 触发守门人占位。
- 守门人 `died` → `RESULT` 结算面板;倒计时归零 → `FAILED`,同面板。

**信号契约(HUD / 面板只订阅,不持状态)**:
```gdscript
signal phase_changed(phase: int)                 # SELECT/RUNNING/GUARDIAN_READY/GUARDIAN/RESULT/FAILED
signal time_changed(remaining: float, total: float)
signal progress_changed(pct: float)
signal pylon_buff_changed(type: String, remaining: float)
# 进度 / 时间来源:连 CombatManager.enemy_killed(enemy, killer, overkill, dir)
#   读 enemy.get_meta("drop_source") 查权重 → add_progress()  (与 LootManager 监听同一信号,互不干扰)
```

> 逐分钟「给老板演示」串场脚本见 **§九**。

## 四、形似强化 · 方向 1:HUD(屏幕常驻战斗界面)

> 现状核对(已读源码):`scenes/ui/hud.tscn` 是空 `CanvasLayer`(layer=100 + 挂 `scripts/ui/hud.gd`),**整棵 Control 树由 `hud.gd._build_ui()` 程序化生成**,已被 `level_*_play.tscn` 实例化。真实数据信号全部就绪。本方向 = **在 `hud.gd` 里扩 `_build_ui()`,不推倒重来**。

### 4.1 设计内容(玩家如何一眼认出「暗黑」)
四块布局,贴合暗黑底部 HUD:

```
 [名牌 Lv·亚马逊]                                          [buff/debuff 图标行]

 (((●)))               [LMB][RMB][ 1 ][ 2 ][ 3 ]                 (((◆)))
 生命球(红)         技能槽位条(中央底部·快捷键角标 / CD扫光)        资源球·专注(蓝金)
   左下             ──────── 经验条(细·弱化 / 满条)────────             右下
```

- **左下生命球**:圆形红球,液面按 HP% 从下往上涨落;顶部高光月牙 + 边缘暗角 + 缓慢浮动波纹线;低血(<30%)叠脉动红光。**暗黑最强形似锚点**。
- **右下资源球·专注**:蓝→金渐变液体(专注 = 本作 mana)。命中回蓝(+4)、被动回蓝(+8/s)时液面肉眼可见「咕嘟」上涨——这个**真联动**细节最卖「割草手感」。
- **中央技能槽位条**:5 格圆角金属边框图标(LMB/RMB/Q/W/E);每格 = 技能占位图标(利 / 多 / 滚 / 冰 / 暴)+ 右下快捷键角标 + 左下专注耗量 + 施放瞬间一道**径向扫光 CD 遮罩**(替换现纯灰蒙板)+ 冷却压暗 + 中央剩余秒数。
- **悬停**:技能图标 hover 弹 tooltip(名称 / 耗专注 / CD / 一句描述,读 `SkillData`);双球 hover 弹「当前 / 上限」。

### 4.2 真功能 vs 纯展示
| 元素 | 定性 | 说明 |
|---|---|---|
| 生命球液面 | ✅真 | 绑 `Player.health_changed` → 设 shader `fill_ratio`(沿用 `_on_health_changed`) |
| 资源球·专注液面 | ✅真 | 绑 `FocusResource.focus_changed`(沿用 `_on_focus_changed`),命中 / 被动回蓝真涨落 |
| 技能槽 CD 扫光 + 压暗 + 秒数 | ✅真 | 绑 `SkillSlotManager.cooldown_changed`,逻辑已在,仅换视觉 |
| 专注不足 → 图标变暗 | ✅真(低成本) | `FocusResource.can_consume(cost)` 每帧判 |
| 球体高光 / 暗角 / 波纹 | 🎭展示 | shader / Tween |
| 经验条 / 名牌 / buff 图标行 / hover tooltip / 低血红光 | 🎭展示 | 经验条本版弱化;buff 行**不接真状态系统** |

> 一句话给老板:**双球和技能 CD 是真的连着血 / 蓝 / 冷却的;名牌、经验条、buff 行是「很像但只为好看」。**

### 4.3 落地要点(Godot4)
- **唯一改动文件 `scripts/ui/hud.gd`**(`hud.tscn` 不动)。三个现有回调全保留,只把「设 `ProgressBar.value`」改成「设球 material `fill_ratio` / 技能槽视觉」。
- **双球(推荐 shader,最省最像)**:`TextureRect` 128×128 + `ShaderMaterial`,新建 `scripts/ui/orb.gdshader`,uniform `fill_ratio:float` / `liquid_color:vec4` + 内置 `TIME`;**一个 shader 复用两次**只换色。**退路**(怕 shader 翻车):圆形液体贴图 + 上层圆遮罩 `clip_children` 调高度做填充比,保 P0 不被卡死。
- **技能槽**:现 `for i in range(5)` 循环里给 `Panel` 套 `StyleBoxFlat`(深底 + 金 / 铜描边 + 圆角);CD 的 `ColorRect` 换 `TextureProgressBar`(`fill_mode = FILL_CLOCKWISE`)做径向扫光。
- ⚠ **mouse_filter 陷阱**:当前整棵树是 `MOUSE_FILTER_IGNORE`(为点地移动穿透)。**只有需 hover 的技能格 / 球改 `STOP`,其余维持 IGNORE**,否则会挡住「左键点地移动」;tooltip 容器本身设 IGNORE。
- 占位图标:无贴图时 `TextureRect.texture = null` + 子 `Label` 单字(利 / 多 / 滚 / 冰 / 暴),留 `# TODO 美术D替图标`。
- 数据源(直接连,勿自造):HP=`Player.health_changed`;专注=autoload `/root/FocusResource`;CD=Player 子节点 `SkillSlotManager`;tooltip=`_slot_mgr.get_skill(i)` 返回 `SkillData`。

新增一张与 `LANDMARKS` 并列的 `STRUCTURES`(`[x, z, kind, rot_deg]`,kind = arch / colonnade / broken_wall / brazier / altar / vault / dais),在 `_rebuild()` 里 `_build_landmarks()` 后加 `_build_structures()`(编辑器 + 运行时都建)。**关键:加一个无碰撞 `_deco_box()`**(纯 `MeshInstance3D`,无 StaticBody / CollisionShape)——装饰件零碰撞、零导航影响、省物理开销;现有 `_box()` 永远建 StaticBody 故装饰件不走它。
- `_build_arch()`:两立柱 + 顶横梁(矩形过梁;半圆拱可用 5 段短梁绕拱顶近似,P1)。
- `_build_colonnade()`:一排细廊柱(`_cylinder` r=0.5)+ 柱头 + 连续顶盖板,柱细 + 无碰撞玩家可穿。
- `_build_brazier()`:盆 + 暖光 `OmniLight3D`,沿连廊每 ~18u 一盏把长廊切成可见推进段。
- `_build_broken_wall()` / `_build_altar()` / `_build_dais()`:断墙 / 发光祭坛 / 视觉高台。
- 八边形精英广场:**从 `boss_room.gd` 搬 `_wall_diag(a,b,mat)`** 做斜边。
- ⚠ 坐标缩放约定:`pos.x/z` 是 WALK 坐标(函数内乘 SCALE),`size` 必须传入时已乘 SCALE,新函数务必沿用,否则装饰件错位。

### 7.4 高低差的诚实做法(避免踩坑)
现 navmesh 是**单层平面**(顶点 `y=0.05`),真实楼梯落差会让 agent 走空 / 穿模。**高低差只做视觉**:`_build_dais` 抬升一块装饰地板(`y=+0.3~0.6`)叠在真实地板上,玩家视觉上「上了台」,但 navmesh / 碰撞地板仍在 `y=0`;凹陷用暗色下沉装饰板 + 无碰撞矮挡边营造「下沉竞技场」感。**真分层 navmesh = P2 砍**(jam 不划算)。
