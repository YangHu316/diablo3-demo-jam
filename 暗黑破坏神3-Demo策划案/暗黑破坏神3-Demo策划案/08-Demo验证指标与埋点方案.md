## 二、验证假设清单(可证伪)

两大验证目标拆为 12 条假设(对应总纲 J 节的 V1~V5 判定)。每条均有量化判据、数据来源、归属目标。**任何一条不写清"用什么数据、卡什么数字证伪",不得进入清单。**

| 编号 | 假设描述 | 量化判据 | 数据来源 | 归属 |
|---|---|---|---|---|
| H1 | 玩家能在目标时间内走完主线精简版 | 主线完成时长中位数落在 **12~18 分钟**(目标约 20min 带内);≥80% 玩家在 20 分钟内击杀屠夫 | N4 漏斗 + `mainline_complete` | 循环(V5) |
| H2 | "首件传奇 ≤8 分钟"的掉落承诺成立 | 首橙时刻中位 ≤6 分钟,P90 ≤8 分钟;≥90% 玩家在 8 分钟内获得首件传奇(脚本软保底+硬保底兜底) | `first_legendary` | 循环(V2) |
| H3 | 传奇掉落能即时转化为构建行为(穿戴率) | 传奇落地 **10 秒内开背包率 ≥80%**;首传奇落地 3 分钟内穿戴率 ≥85% | `legendary_moment` + `ui_open` + `item_equip` | 循环(V2/V3) |
| H4 | 穿戴传奇/换 Build 能被感知为"变强" | ①节点脉冲打分(变强感)均值 ≥4.0;②拿到「专注引擎」后施法密度(`skill_usage_summary` 的 cast/分钟)较拿到前提升 ≥20%;③穿戴后 3 分钟 kpm 较前 3 分钟提升 ≥25% | `pulse_response` + `skill_usage_summary` + `kill_summary` | 循环/手感(V3) |
| H5 | "变强后想打更难的"成立(进梦魇率) | 击杀屠夫的玩家中 **≥75%** 在无强制引导下主动进入梦魇挑战 | N4→N5 转化率 | 循环(V4) |
| H6 | 梦魇 2 层具备通关与重复吸引力 | 进入梦魇者 **推进至第 2 层 ≥75%、通关第 2 层 ≥50%**;通关者中 ≥25% 主动重复挑战至少 1 次 | N5→N6 + `nightmare_retry` | 循环(V4) |
| H7 | 弓系打击感达到"爽"的基线 | 问卷"打击感"5 分制均值 **≥4.0**,且 ≤2 分占比 ≤10% | 问卷 Q1 | 手感(V1) |
| H8 | 技能释放具备爽感且构筑自由被实际使用 | 问卷"技能爽感"均值 ≥4.0;**1~8 级期间技能/符文主动更换次数中位数 ≥3 次**;全程主动换装中位 ≥4 次 | 问卷 Q2 + `skill_change` + `item_equip` | 手感(V1/V3) |
| H9 | 30 分钟体验黏性足以支撑连续游玩 | 平均连续游玩时长 ≥28 分钟(Demo 总内容约 30min);中途主动退出率(N1 后、N4 前退出且未回归)≤15% | `session_heartbeat` + `app_quit` | 手感(V1) |
| H10 | 升级节奏无断档(每级必有新东西) | 1~8 级**每级均触发 ≥1 个解锁**(`skill_unlock`/符文/被动/槽位);任意相邻两级间隔中位 ≤3.5 分钟;**任意 3 分钟窗口无"新东西"占比 <10%** | `level_up` + `skill_unlock` + 节拍推导 | 循环(V5) |
| H11 | 难度曲线"有挑战不挫败" | 全程人均死亡 1~4 次;**屠夫首次挑战失败率落在 30%~60%**;任一非 Boss 区域死亡占比 ≤25% | `player_death` 位置热区 + `boss_fight_*` | 手感(V1) |
| H12 | **职业内不同 Build 体验均衡,结论不被单一 Build 偏置** | 三套弓 Build(A 多重散射 / B 冰控精准 / C 机动游击)在 H7 打分差 ≤0.4 分、在屠夫失败率差 ≤15 个百分点;无任一 Build 被 <10% 玩家采用 | 全量数据按 `build_tag` 分组 | 手感(V1) |

> **证伪即有效**:任何一条假设被推翻,都必须指向第六章的回炉决策,而不是修改阈值。阈值调整须经制作人+数据策划双签,且只允许在测试开始前。
> **一票否决**:对应总纲 V1 与 V4 —— 手感目标(H7/H8/H11)与循环目标(H5/H6)任一不达标,即判验证失败,不接受跨目标互救。

## 三、核心指标体系

### 3.1 主漏斗与目标转化率

漏斗按"邀约制 playtest"口径设定(受邀玩家完成意愿高于自然流量,故阈值从严)。注意 N1 为**无选职业的直接创建**,故 N0→N1 几乎无设计性流失,主要看客户端稳定性。

| 漏斗段 | 目标转化率(逐段) | 红线(低于即重大警报) | 主要流失归因方向 |
|---|---|---|---|
| N0 启动 → N1 创建角色 | ≥98% | <95% | 客户端稳定性、首屏到进入 |
| N1 创建角色 → N2 出城门 | ≥95% | <90% | 开场 2 分钟移动/射击教学与节奏 |
| N2 出城门 → N3 救凯恩 | ≥92% | <85% | 寻路引导、大教堂段战斗密度 |
| N3 救凯恩 → N4 屠夫击杀 | ≥85% | <75% | 中段时长、难度曲线、地穴段/痛苦大厅节奏 |
| N4 屠夫击杀 → N5 进梦魇 | **≥75%**(自然转化,无强制) | <50% | 梦魇入口包装与动机设计 |
| N5 进梦魇 → N6 二层通关 | **≥50%**(通关);推进至 2 层 ≥75% | <40% | 梦魇 2 层难度斜率(×2.2 生命/×1.5 伤害) |
| **N0 → N6 全程完成率** | **≥45%** | **<28%** | 综合 |

辅助口径:每个节点同时统计**到达时刻分布**(P25/P50/P75),与第 3.2 节节奏指标交叉验证。屠夫击杀(N4)到达时刻是 H1 的核心锚点,目标 P50 落在 17~20 分钟。

### 3.2 节奏指标

| 指标 | 定义 | 目标 | 数据来源 |
|---|---|---|---|
| 升级间隔分布 | 每级 `level_up` 间隔的 P50/P90 | P50 全段 ≤3.5 分钟;前 3 级每级 ≤2.5 分钟;P90 任意一级 ≤5 分钟 | `level_up` |
| 每级解锁覆盖 | 每个 `level_up` 后 90 秒内是否伴随 ≥1 个 `skill_unlock`/符文/被动/槽位 | 1~8 级每级 100% 覆盖 | `level_up` + `skill_unlock` |
| "新东西"窗口空白率 | 任意滑动 3 分钟窗口内无新解锁/新装备/新机制/新区域的占比 | **<10%**(对应 V5);全程"空窗"≤1 处 | 多事件推导 |
| 首传奇时刻分布 | `first_legendary` 的 play_time 分布 | **P50 ≤6 分钟,P90 ≤8 分钟,最大值 ≤8 分钟**(硬保底兜底) | `first_legendary` |
| 首传奇→开背包延迟 | 橙光落地到 `ui_open(inventory)` 的时间差 | **10 秒内开背包率 ≥80%** | D 组事件 + F1 |
| 技能/符文更换次数 | 1~8 级 `skill_change` 主动次数(不含解锁自动上槽) | 中位数 ≥3;为 0 的玩家占比 ≤20% | `skill_change` |
| 主动换装次数 | 全程 `item_equip` 中主动更换(非首次空槽穿戴)次数 | 中位 ≥4 | `item_equip` |
| 死亡次数与热区 | 人均死亡数;按 map_id+坐标聚类的死亡热区 | 人均 1~4 次;热区 Top1 若为非 Boss 区域且占比 >25%,标记为难度毛刺 | `player_death` |
| 屠夫战时长与失败 | 首杀尝试次数、单次战斗时长 | 单次战斗 **100~150 秒**(TTK 目标);首战失败率 30%~60% | `boss_fight_*` |
| TTK 锯齿校验 | 白怪单体主要技能命中发数 | 全程 2.0~3.3 发;Lv4(换 T2 武器)/Lv7(换 T3 武器)瞬间回落≈2 发 | `skill_usage_summary` + `kill_summary` |

### 3.3 手感指标(主观 + 客观代理)

| 类别 | 指标 | 目标 |
|---|---|---|
| 主观 | 问卷 Q1 打击感(5 分制) | 均值 ≥4.0,≤2 分占比 ≤10% |
| 主观 | 问卷 Q2 技能爽感(5 分制) | 均值 ≥4.0 |
| 主观 | 节点脉冲(屠夫战后即时 1 题:"刚才这场 Boss 战爽吗") | 均值 ≥4.0 |
| 客观代理 | 平均连续游玩时长 | ≥28 分钟(Demo 总内容约 30 分钟) |
| 客观代理 | 主动重复挑战梦魇次数 | 通关者人均 ≥0.3 次;重复率 ≥25% |

## 四、埋点事件全表(交付程序)

### 4.0 全局规范

- **公共参数**(所有事件自动携带,事件表内不再重复):`event_id`、`player_id`(设备级匿名哈希)、`session_id`、`build_version`、`char_level`、`play_time_s`(本存档累计游玩秒数)、`map_id`、`nightmare_layer`(非梦魇为 0,取值 1/2)、`build_tag`(当前主导 Build:A_spread/B_freeze/C_mobile/none)、`client_ts`(毫秒)。
- **职业固定**:Demo 为单职业,`class` 恒为 `amazon`,**不作为公共参数下发、不参与任何分组**;凡历史方案中的 `class_id` 分组口径一律作废。
- **落盘与上报**:单机本地 JSONL 逐条落盘(防崩溃丢数据),会话结束或心跳时批量上报;无网环境随包回收本地文件。
- **聚合规则**:标注【聚合】的高频事件在客户端按 30 秒窗口汇总上报,严禁逐条发送;其余事件逐条实时。
- **坐标**:`pos_x/pos_y` 为地图局部坐标,配合 `map_id` 还原热区。
- **验收**:QA 按本表逐条验收触发时机与参数完整性,缺参视为埋点 Bug。

### A. 会话与系统(4)

| # | 事件名 | 触发时机 | 携带参数 | 用途 |
|---|---|---|---|---|
| A1 | `app_start` | 客户端启动完成进主菜单 | `device_spec`(分级)、`is_first_launch` | 漏斗 N0;到达基数 |
| A2 | `app_quit` | 玩家退出或进程结束 | `quit_context`(menu/town/combat/cutscene/nightmare)、`session_duration_s` | H9 退出率与退出点分布 |
| A3 | `session_heartbeat` | 每 60 秒 | `fps_avg`、`fps_p1_low`、`afk_flag` | 连续游玩时长;性能对手感评分的干扰排查 |
| A4 | `client_error` | 崩溃/卡死/脚本异常 | `error_code`、`stack_hash` | 稳定性;剔除受污染样本 |

## 六、成功 / 失败 / 灰色区判定与决策

### 6.1 判定结构

两大验证目标分别裁决,**互不抵扣**(手感满分救不了循环崩盘,反之亦然)。**一票否决**:对应总纲——V1(手感)与 V4(循环)任一不达标即整体验证失败。

**目标一:弓系战斗手感(关键假设 H7、H8、H11、H12)**

| 区间 | 判定条件 | 决策 |
|---|---|---|
| 成功(绿) | H7、H8 全过,H11 不触红线,H12 不出现单 Build 失衡 | 手感方案冻结,弓系参数表(取消后摇窗口/僵直阈值/穿透命中表现/冰冻碎裂)与三套 Build 进入立项基线 |
| 灰色 | Q1 或 Q2 均值落在 3.5~4.0,无红线 | 锁定低分子项(按 `skill_id`×`build_tag` 拆 C1 数据),2 周手感专项迭代后**只重测手感**(小样本 15 人即可) |
| 失败(红) | Q1 或 Q2 <3.5,或 ≤2 分占比 >20%,或中途战斗内退出率 >25% | 03-战斗案反馈分层参数回炉,垂直切片不进入立项评审 |

**目标二:30 分钟循环(关键假设 H1~H6、H10)**

| 区间 | 判定条件 | 决策 |
|---|---|---|
| 成功(绿) | H1、H2、H3、H5 全过,H6 不触红线,H10 每级解锁 100% 覆盖 | 核心循环验证通过,掉落表(5 传奇/10 词缀)与梦魇 2 层结构进入立项基线 |
| 灰色 | 1~2 条关键假设落在目标与红线之间 | 按 6.2 归因表定位单模块回炉,2 周内复测全流程 |
| 失败(红) | H1 中位 >22 分钟 或 H2 首橙 P90 >8 分钟 或 H5<50%,或任意两条关键假设触红线 | 循环结构性失败,回总纲层重审 Demo 切片结构(时长/掉落预算/梦魇入口),禁止用调参粉饰 |

**立项推进条件:两个目标同时绿,或一绿一灰且灰区已有明确归因与修复方案。任一红 = No-Go。V1 与 V4 任一红线即一票否决。**

### 7.3 主持人脚本要点

1. **开场(2 分钟)**:只说"这是早期测试版本,测的是游戏不是你;遇到问题是我们的素材,不是你的失误"。**不剧透任何目标时长、不提'8 分钟传奇'等承诺、不暗示有几套 Build**,避免污染节奏与构建感知。
2. **出声思考**:鼓励玩家边玩边说,但主持人不追问"你觉得打击感怎样"这类引导性问题;只允许中性追问("刚才发生了什么?")。
3. **介入规则**:玩家卡关 ≥3 分钟且明显烦躁才可提示,提示内容与时点记入观察表(该样本的对应漏斗段打"受助"标记,转化率统计中单列)。
4. **不救场**:玩家想中途退出,允许并记录原因,**不挽留**——退出本身是 H9 的数据。
5. **结束访谈(问卷后,4 分钟,3 问)**:①"用一个词形容刚才的弓箭战斗";②"还记得捡到第一件橙装是什么时候吗?当时在干嘛"(测掉落记忆度,核对 ≤8min);③"进梦魇之后,你换没换技能或装备?为什么"(测 H3/H8 构建动机)。
6. **场后 24 小时内**:观察表电子化归档,与该玩家埋点数据按 `player_id` 对齐入库。
