## 二、Demo 定位与设计总纲

### 2.1 一句话定位

**一个 30 分钟的单机垂直切片,单职业亚马逊弓系远程**,只回答两个立项级问题:

1. **弓系远程核心战斗手感是否"爽"**——支柱 S1(即时爽快的远程屠杀)在最小实现下是否成立,且**职业内不同 Build 的手感差异是否能被玩家感知**;
2. **"掉落→构建→挑战更难内容"的核心循环能否在 30 分钟内成立**——支柱 S2/S3/S4 的最小闭环能否被普通玩家自发跑通。

Demo 不是"小号正式版",而是**针对这两个问题的实验装置**。一切内容取舍以"是否服务验证"为唯一标尺。30 分钟 = **主线 20min(打到屠夫)+ 梦魇挑战 10min(2 层)**。

### 2.2 全组统一约束(总纲,各分册不得突破)

> 本表为全分册唯一事实源。任何分册内容与本表冲突,**一律以本表为准并就地改正**。

| # | 约束项 | 规定 |
|---|---|---|
| G1 | 体验时长 | 总 **30 分钟** = 主线 **20 分钟**(到屠夫)+ 梦魇挑战 **10 分钟**(2 层) |
| G2 | 剧情范围 | 仅第一幕精简版:新崔斯特姆 → 受难大教堂救迪卡·凯恩 → 痛苦大厅 → 屠夫 Boss 战收尾 → 泰瑞尔觉醒;保留莉娅、凯恩、泰瑞尔核心节点;**凯恩存活**(不死);仅保留 1 个随机事件(灵魂之瓮);**3 任务对应 3 关卡场景块** |
| G3 | 职业 | **仅亚马逊 1 职业,单性别**;弓系远程;主属性 = **敏捷**;资源 = **专注**(回复-爆发型,上限 100,被动回复 8/秒,利箭命中回 4);主动技能仅 5 个 |
| G4 | 等级 | **1~8 级**;**每级必有新解锁**(技能/符文/被动);**技能槽 = 5**(左键/右键 + 3 数字键),**第 5 级即全部开放**(满足"6 级前全开");**不存在第 6 槽**(解锁节奏见 4.2) |
| G5 | 掉落节奏 | 首件传奇 **≤8 分钟**(脚本保底);传奇掉率约为正式版 5 倍;词缀池 **10 条**;品质 **3 档**(魔法/稀有/传奇);传奇 **5 件**;获取仅"掉落+保底"一条管线;智能掉落 **100% 定向敏捷**(单职业);无主副词缀分轨 |
| G6 | 难度 | 单一"普通"难度 + 数值动态微调;通关屠夫后解锁"梦魇挑战"(**2 层**递进);梦魇**不做新地图、不做新 Boss、不做新美术**,100% 复用关卡 2(大教堂)/关卡 3(痛苦大厅)的 Tile,仅做数值缩放 + 精英/密度调整 |
| G7 | 砍掉系统 | 制造与经济(仅极简金币回收)、巅峰、赛季、多人与社交、交易全部不做;随从仅保留圣堂武士科尔玛(剧情需要,功能简化:跟随+普攻+10 秒治疗);图鉴/成就降为**加分项**(P2,有余力再做,不占 P0/P1,不作为验证依赖) |
| G8 | 节奏原则 | 玩家**每 2~3 分钟必有一次"新东西"**(新技能/新怪/传奇掉落/剧情演出),此为节拍轴(第六章)的验收红线;首件传奇 ≤8 分钟 |

> **唯一 Boss 声明**:全 Demo 唯一 Boss = **屠夫**。原骷髅王 Boss **取消**;其"中段教学战"职能由大教堂地穴段的黄名精英"**地穴守卫**"承担(**非 Boss、不计 Boss 数**)。梦魇第 2 层守门"**梦魇屠夫**"= 屠夫的数值调整复用(非新 Boss)。

### 2.3 为什么这样裁:三条裁剪逻辑

完整版核心循环为"战斗→掉落→构建→更难内容→反哺产出"。Demo 的全部裁剪服从三条逻辑:

1. **循环四环节一个都不能少,但每个环节只留最小可验证实现**。掉落留 3 档品质,因为验证"橙光多巴胺"只需"普通—较好—惊喜"三级对比;难度留"普通 + 梦魇 2 层",因为验证"更难内容"只需一条 2 级阶梯,不需要多档滑条;职业只留亚马逊 1 个,因为验证"弓系手感 + 职业内 Build 差异"不需要第二个职业。
2. **凡是服务"千小时长线"的系统全部砍掉**。巅峰、赛季、制造、交易解决的都是 50 小时后的问题;留着只会稀释开发资源、污染测试数据(玩家精力被分流到非验证项)。图鉴/成就同理降级为加分项,不进 P0/P1。
3. **凡是拖慢节奏的环节一律压缩,且压缩本身就是实验变量**。Demo 把首橙提前到 8 分钟内、掉率 ×5,本质是把"小时级循环"压缩成"分钟级循环"——**要验证的正是压缩后的循环曲线是否依然成瘾**,数据回传后反推正式版投放节奏。同理,30 分钟内 1~8 级、每级必有新解锁,是把成长节奏压到"每 2~3 分钟一次新东西"的极限。

## 三、验证目标与非目标

### 3.1 验证目标(全部可量化,判定标准见第八章)

| 编号 | 验证目标 | 对应完整版支柱 | 验证手段 |
|---|---|---|---|
| V1 | **弓系远程手感**:点击→击杀→拾取的即时反馈是否"爽"(亚马逊弓系);**且职业内不同 Build 的手感差异是否成立**(多重散射 / 冰控精准 / 机动游击) | S1 | 问卷打分 + 录屏行为分析 |
| V2 | 掉落驱动:橙光落地是否构成可观测的兴奋事件;首件传奇是否 ≤8 分钟;玩家是否因掉落主动停下检视装备 | S2 | 埋点(掉落后 10 秒内打开背包率)+ 首橙时刻统计 + 问卷 |
| V3 | 构建行为:玩家是否**自发**(非引导强制)更换技能/符文/装备以应对更难内容 | S3 | 埋点(主动换装/换技能/换符文次数) |
| V4 | 循环成立:玩家通关屠夫后是否主动进入梦魇挑战,并在 2 层递进中感知"变强→打更难"的正反馈 | S4 | 梦魇进入率 / 层数推进率 / 通关率 / 问卷 |
| V5 | 节奏压缩可行性:2~3 分钟压缩后的解锁与掉落节奏,玩家是"兴奋"还是"麻木"(奖励通胀阈值探测) | S1–S4 | 分段问卷 + 录屏"新东西"时间戳统计 |

> **V1 口径变更说明(V2.0)**:删除原"近战/远程双职业对比"。本 Demo 为单职业亚马逊弓系,V1 改为验证**弓系远程手感本身是否爽**,叠加**职业内 Build 差异**(同一职业、不同技能/符文/核心传奇件组合,手感是否能被玩家明显区分)。

### 3.2 非目标(明确不验证什么——同样是全组纪律)

| 编号 | 非目标 | 说明与纪律 |
|---|---|---|
| N1 | 不验证剧情质量与叙事节奏 | 剧情仅作为流程容器与情绪标点;演出做到"能看懂"即可,不追求正式版演出规格。**禁止文案侧借 Demo 打磨台词与支线** |
| N2 | 不验证长线留存与经济平衡 | 无赛季/巅峰/制造,任何"第 1 小时之后"的问题不在本 Demo 回答范围 |
| N3 | 不验证多人体验 | 纯单机;组队收益、个人化掉落等规则一概不实现 |
| N4 | 不验证美术品质与世界观沉浸感 | 允许使用代理资产与复用拼块;测试问卷明示"美术非评价项"(见风险 R7) |
| N5 | **不验证职业间平衡** | **单职业 Demo,先天无法做职业间 DPS 对齐**;只验证"亚马逊弓系手感自身成立 + 职业内 Build 差异成立",职业间平衡留给正式版(完整版 02 分册) |
| N6 | 不验证手柄操作 | 仅键鼠;双范式同权留给正式版(完整版 10 分册) |
| N7 | 不验证多档难度曲线 | 梦魇 2 层只验证"递进阶梯是否驱动构建",不验证指数曲线参数本身 |

> **纪律条款**:任何人提出的新增需求,若不能映射到 V1–V5,一律进入"正式版需求池",不进 Demo。仲裁人:主策划。

### 4.3 难度动态微调规则(G6 的执行口径,05 分册细化)

- 监测窗口:滚动 3 分钟内玩家死亡次数与击杀耗时中位数;
- 上调(玩家碾压):怪物血量 +10%/档,最多 +30%;下调(玩家受挫):怪物伤害 −15%/档,最多 −30%,药水冷却 −20%;
- 调整不提示、不出 UI——Demo 要的是"所有测试者都落在可比较的体验带宽内",不是难度系统本身;
- 梦魇挑战 2 层**关闭动态微调**(层差就是实验变量,必须固定);屠夫战失败门前检查点满血重试,固定 5s 不递增。

## 十一、结语:Demo 与正式版的关系

验证通过后**直接进入正式版生产线的资产**:战斗管线与反馈分层规格(03)、梦魇数值缩放器与复用框架(06)、配置表管线、掉落表现规格(10)、亚马逊弓系 5 技能与符文机制(02)。

**回传修正完整版分册的数据**:掉落投放曲线(R2 对照组 → 04/05)、解锁节奏密度(4.2 实测 → 02 `unlock_schedule`)、密度节拍参数(R8 跑测 → 03/11)、弓系手感参数(V1 → 03)。

**失败同样有价值**:V1 失败即弓系反馈规格推翻重做,V4 失败即"掉落→构建→更难内容"循环最小形态不成立——都必须在多职业、五幕量产前知道答案。本 Demo 用 30 分钟、单职业、最小资产,把立项最核心的两个赌注先押一遍。
