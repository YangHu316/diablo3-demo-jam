### 本文档的三条铁律(与 Demo 总纲对齐)

1. **被砍系统的入口在 UI 上一律不出现**——不做灰化、不做"敬请期待",直接不存在。playtest 用户的注意力是采样资源,30 分钟极其稀缺,不许浪费在死按钮上(职业选择、巅峰、仓库、商店、工匠、聊天、世界图、成就一律不出现)。
2. **禁止超过 15 秒的强制弹窗教学**。完整版的"模态教学卡"在 Demo 中仅保留 2 处(**首次死亡、梦魇规则**),其余全部降级为高亮指引或 Toast。前 8 分钟**零模态**。
3. **每 2~3 分钟一次"新东西",每个新东西必须有一次醒目 UI 反馈**。反馈优先级与仲裁规则见第 5 章——同屏只允许一个"主角演出"。每级必有新解锁(槽/技能/符文/被动),每次解锁都是一记节奏鼓点。

## 1. 设计目标:UI 是验证三问题的测量仪器

Demo UI 不是完整版 UI 的删减版,而是把《00/J》五条验证目标转译成"看得见、可埋点"的界面信号。本分册直接对接 V1/V2/V3/V4/V5 的判定阈值:

| 验证目标(来自 J) | UI 侧职责 | 验收信号(界面可观测) |
|---|---|---|
| **V1 手感**:弓系远程爽不爽 + 职业内 Build 手感差异 | 弓系命中反馈(穿透/暴击黄字/冰冻碎裂)、技能栏 5 槽即点即射、换技能/换符文界面顺滑 | 问卷打击/技能爽快感均值 ≥4.0 且 ≤2 分占比 ≤10% |
| **V2 掉落**:橙光是否构成可观测兴奋事件 + 首传奇 ≤8min | 传奇光柱/星标/音效分级反馈;首橙脚本保底节点的演出 | 传奇落地 **10 秒内开背包率 ≥80%**;首橙时刻中位 ≤6min、P90 ≤8min |
| **V3 构建**:是否自发换技能/符文/装备应对更难 | 对比 Tooltip(三围 delta)、技能/符文/被动配置界面、回城构建引导 | 全程主动换装中位 ≥4 次;进梦魇前后调整技能/符文玩家比例 ≥60% |
| **V4 循环**:屠夫后是否主动进梦魇、感知"变强→打更难" | 梦魇裂隙入口、进层横幅、层内计时条、层结算页 | 屠夫后主动进梦魇 ≥75%;梦魇 2 层通关率 ≥50%、推进至 2 层 ≥75% |
| **V5 节奏**:2-3 分钟压缩节奏兴奋还是麻木 | 升级金光、解锁角标、掉落演出构成的"鼓点序列";引导超时埋点 | 任意 3 分钟窗口无"新东西"占比 <10%;主线完成中位 12-18min |

> **一票否决提醒**:V1 与 V4 任一不达标即判验证失败。因此本分册把最高打磨预算压在"弓系命中反馈链"(服务 V1)与"梦魇入口/结算/进层横幅"(服务 V4)两处,绝不为次要 QoL 分散资源。

## 4. 新手引导分钟级脚本(0~8 分钟,零模态)

### 4.1 设计方法

- **引导形式只用两档**:`Toast`(右下角,不打断,8s 淡出)与 `高亮`(目标 UI 金框 + 箭头 + ≤24 字文案,不锁操作,完成即灭)。**0~8 分钟内零模态卡**(模态卡只用于首次死亡与梦魇规则,均在 8 分钟后)。
- **为什么压缩到前 8 分钟**:30 分钟切片里,前 8 分钟必须完成"弓系射击/拾取/对比穿戴/换技能/首件传奇"的全部核心教学,且首件传奇脚本保底落在 ≤8min(服务 V2)。8 分钟后玩家应已会全部基础循环,引导转入"粗粒度节点",把节奏让给战斗与构建。
- **每条引导绑定"完成判定"与"超时兜底"**:玩家自发完成则引导静默跳过(不奖励不打扰);超时未完成则升一档重发一次,之后永久沉默——绝不重复唠叨。
- **关卡配合脚本化**:首件蓝装、首瓶药水机制、首次掉血遭遇战、首件传奇均为关卡脚本保底投放(与《H 关卡》《F 掉落》对齐),保证引导触发时机方差 <±45 秒(30min 切片对时序更敏感)。

| 时间窗 | 触发时机 | 引导形式 | 教学内容 | 完成判定 | 超时兜底 |
|---|---|---|---|---|---|

| 1:30~2:30 | 首件蓝装保底掉落(脚本固定,本职业弓,DPS+60%) | 高亮(地面标签) | "踩上去/点击拾取装备" | 拾取入包 | 20s 未拾:小地图白点 + Toast 重发 |

| 3:00~4:00 | 协防城门第 2 波尸潮(脚本保底使 HP 首次 <60%);治疗药水默认在槽 | 高亮(药水槽) | "血量告急!按 Q 喝药(回 60% 生命)" | 饮用 1 次 | HP<40% 时血球红光脉冲 + 文案加大一档;此战怪物伤害有软上限,不致死 |

| 6:00~7:00 | 升 4 级,**符文系统开启**(首批符文)+ 翻滚回避(槽 4)解锁 | Toast + 技能界面符文页红点 | "符文=技能的免费变体;空格=翻滚回避(0.4s 无敌帧)" | 在符文页切一次符文 **或** 战场用一次翻滚 | 不兜底(进阶项);翻滚槽持续呼吸光提醒位移可用 |
| 7:00~8:00 | **首件传奇脚本保底掉落(≤8min 硬保底)**,大教堂段内 | 纯反馈 + 高亮 | **橙柱 + 低音锣 + 0.3s 震屏**;拾取后高亮橙字特效文本 3s(无鉴定流程,Demo 无未鉴定态) | **埋点:落地→开背包耗时(V2 核心采样:目标 10s 内 ≥80%)** | 若 8min 前任何保底载体未出橙,**下一个保底载体强制 100% 出橙**(首橙白名单:女妖之弓/冰霜之箭袋/疾风之靴) |

> **设计校验**:本表 8 条把"移动→射击→拾取→对比穿戴→第二技能→喝药→进技能界面装/换技能→符文+翻滚→首传奇"一次走完,恰好覆盖 V1/V2/V3 三条验证目标的全部基础动作,且首橙锚定 ≤8min。

## 5. 反馈包装规范(技能解锁与装备掉落)

### 5.1 反馈优先级表

同一时刻(±0.5s 窗口)多个反馈竞争时,**视觉演出只播最高优先级一个,音效按"高优先级全量、低优先级音量 -6dB"混音**;被压制的视觉反馈转为静默常驻态(光柱/星标本就常驻,不丢信息)。

| 优先级 | 事件 | 视觉 | 音效 | 时长 |
|---|---|---|---|---|
| **P1** | **传奇掉落** | 粗橙光柱(高 6m,**不拾取永不消失**)+ 小地图橙星 + 0.3s 震屏 | 专属低音锣 | 演出 1.0s,光柱常驻 |
| P2 | 升级(L1→L8 共 8 次) | 全屏金光柱 + 生命/专注回满波纹 | 升级号角 | 1.2s,不锁操作 |
| P3 | 技能槽/符文/被动解锁 | 对应槽位金光绽放 + "!"角标(进入技能界面前不灭) | 短促圣光音 | 0.8s |
| P4 | 稀有掉落 | 短黄光柱(3s 衰减) | 专属"叮" | — |
| P5 | 药水冷却就绪 / 女武神大招就绪 / 专注满 | 槽位/资源球白闪 | 轻"咔" | 0.3s |
| P6 | 负反馈(专注不足/CD 未好) | 槽位暗闪 + 资源球红边短闪 | 弓弦空响(-6dB) | ≤0.2s |

### 5.2 Demo 特化规则

1. **首次事件加倍**:首件传奇、首次升级、首个技能解锁、首次橙字特效,在标准演出外追加一行屏幕下沿居中飘字(如"传奇物品!"),仅第一次,之后不再。
2. **演出排队**:升级与掉落同帧发生时(击杀屠夫/梦魇屠夫常见),传奇演出先行,升级金光延迟 1.0s 播放——宁可迟到,不许叠加。同样适用于"屠夫首杀必掉传奇 + 击杀时升级"的高潮帧。
3. **演出锚点禁投掉落**:剧情演出节点前后(开场陨星动漫、莉娅初见、救凯恩 IGC、屠夫登场"鲜肉!"、泰瑞尔觉醒)禁止投放掉落,避免橙光被演出黑边吞掉。
4. **棘轮效应教训**:**Demo 的演出强度就是正式版基准**,问卷将直接询问"掉落演出是否过弱/过强",首发宁可保守半档。
5. **梦魇进层横幅外显**:进入梦魇每层时全屏横幅(层名 + 强度倍率 + "橙权重提升"提示),复用掉落光柱管线,成本≈0,直接服务 V4 的"感知变强→打更难"。

## 6. 物品 Tooltip 精简规范(4 区块)

单一渲染组件,地面悬停/背包共用。完整版 14 字段收口为 **4 个区块**:

| 区块 | 内容 | 说明 |
|---|---|---|
| **区块 1 标题区** | 物品名(品质色)+ 类型部位 + 需求等级 | 品质 3 档:魔法=蓝、稀有=黄、传奇=橙;弓系定向掉落,需求等级保留红字防呆 |
| **区块 2 核心数值区** | 核心大数字(武器 DPS / 防具护甲)+ 明细 | 武器显示 Tier 对应 DPS 区间(T1 3-8 / T2 9-18 / T3 20-32) |
| **区块 3 词缀区** | 词缀列表(来自 10 条词缀池)+ 传奇橙字特效(不占词缀位) | 魔法 1-2 词缀 / 稀有 3-4 词缀 / 传奇 4 词缀 + 橙字特效(≤40 字);无主副词缀分轨 |
| **区块 4 对比区** | **对比 Tooltip + 三围 delta(必须保留,不可降级)** | 悬停即并排弹"已装备"对比;伤害/韧性两行必显(绿↑红↓百分比箭头),恢复行仅当涉及恢复词缀时显示;传奇特效行附"特效价值需实战体会" |

**砍掉的字段(及裁剪理由)**:
- 远古/太古徽记 → 品质只到传奇,无此档。
- 套装区 → Demo 无套装。
- 插槽镶嵌 → 无宝石系统。
- 风味文本 → 缩短阅读路径,服务"3 秒决策"。
- 出售/分解预览 → 经济极简,无分解/出售。
- Ctrl 查看 [min-max] 区间 / 高级 Tooltip 开关 → 高级信息,8 级无场景。

> **验收口径**:任意掉落,玩家从悬停到做出穿/不穿决策 ≤3 秒;对比箭头是 Demo 教学量最低、信息效率最高的单个 UI 元素,P0 中的 P0,直接喂给 V3。

## 9. 入场确认与结尾问卷(Demo 专属)

### 9.1 亚马逊立绘确认入场(替代选职业界面)

> **裁剪理由**:Demo 是**单职业垂直切片**,职业不是变量,做"选职业界面"既无意义又浪费玩家注意力,且 V 系列验证里已**去掉"职业对比"维度**。因此**砍掉选职业界面**,改为单张亚马逊立绘的极简确认入场。

- **形式**:开场动态漫画(≤30s,可跳过)后,单屏亚马逊立绘 + 一句话定位("**精准弓手,专注爆发——3 秒倾泻,数秒回填**")+ 复杂度提示(弓系远程,主属性敏捷,资源专注)。
- **交互**:**无捏脸、无命名、无性别选择、无职业切换**。底部 [开始冒险] 单按钮 → 直接入场,全程 ≤10 秒、≤1 次点击。亦可做成"无确认直接入场"——动漫结束即操控亚马逊,立绘仅 1.5s 一闪而过(取更激进方案,把时间还给战斗)。
- 立绘旁播放 3s 弓系战斗循环(引擎内录屏:利箭穿透 + 多重射击扇形),帮玩家第一眼建立"弓系手感"预期(服务 V1 的首因效应)。

### 9.2 结尾问卷/愿望单

- **触发**:通关梦魇第 2 层结算后(主路径,经"Demo 数据回放"自然过渡),或游戏时长 ≥30 分钟后玩家按 ESC 退出时(兜底路径,弹出一次,可拒绝)。
- **形式**:全屏分页,**5 题封顶**,每页 1 题大按钮制,30 秒可答完:
  1. **核心战斗爽不爽**(5 分制)——弓系远程命中→击杀→拾取的即时反馈。【直接喂 V1,均值 ≥4.0 阈值】
  2. **不同 Build 手感差异**(5 分制)——多重散射 / 冰控精准 / 机动游击 之间是否感受到差别。【V1 的"职业内构建差异",**替代原"职业对比"题**】
  3. **掉落兴奋度**(5 分制)——橙光是否构成兴奋事件。【喂 V2】
  4. **引导是否打扰**(3 选 1:几乎无感 / 恰到好处 / 太啰嗦)。【喂 V5】
  5. **最想要的下个内容(愿望单复选)**:更多职业 / 更多 Build 与符文 / 更高难度梦魇 / 制造与仓库系统 / 多人 / 手柄支持 / 图鉴成就。【**替代原职业对比题**,用真实需求数据决定正式版优先级】
  - (可选第 6 题:一句话留言,选填。)
- 提交后显示制作组致谢页 + 复用 8.2 的"Demo 数据回放"作为情绪奖励。**不答也能正常退出**。

> **去职业对比题的理由**:单职业 Demo 没有横向职业可比;把那道题的预算转给"**Build 手感差异**"(V1 内涵)与"**最想要的下个内容愿望单**"(为正式版排序),信息密度更高。

## 10. 手柄支持建议:Demo 只做键鼠

**结论:不做手柄,且这是收益最高的一刀。** 理由:

1. **工期**:完整版已评估"双范式同权导致 UI 工期 ×1.6",焦点系统框架是最大单项。Demo 砍手柄≈节省 UI 侧 40% 工期,直接换来弓系命中反馈链与引导脚本的打磨人力——而这两处恰是 V1/V5 的命门。
2. **范式不可半做**:暗黑3 主机版的成功恰恰证明手柄版必须整套重设计(直接操控 / 自动瞄准 / 闪避 / 列表背包)。**弓系远程对自动瞄准尤其敏感**:Demo 若做"缩水手柄版",采到的将是对一个不存在产品的差评,污染验证数据。
3. **自动瞄准影响数值结论**:手柄 ±30° 自动瞄准会改变弓系命中难度与"爽不爽"手感,双输入混采让"战斗爽不爽"问卷数据不可归因(V1 一票否决项,绝不能让它沾上噪声)。
4. **playtest 场景**:内部与小规模外测均为 PC 环境,键鼠覆盖 100% 受众。

**留两条后路**:① 全部界面布局沿用完整版"焦点可达"网格规范设计(只是不实现焦点系统),保证正式版接手柄时不需重排版;② 问卷愿望单中放入"手柄支持"选项,用真实需求数据决定正式版手柄优先级。
