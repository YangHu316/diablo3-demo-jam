# 暗黑 Demo · Gamejam 重切方案与分工 v2(Godot / 亚马逊弓箭手 / 4 人 / 2 天)

> v2 取代 v1。更新点:引擎锁 **Godot**、素材用 **3D 做 2.5D**(全部自购)、职业改 **亚马逊(弓箭手·远程)**、保留 **1 随从 + 1 对话 NPC**、分工改为 **战斗/系统/关卡/角色** 四域、新增 **Claude 全流程主导** 一节(第 8 节)。
> 核心回路不变:点击射击 → 击杀(爽)→ 掉落(橙光)→ 换装变强 → 挑战更难内容。

## 0. 技术与设计前置(已按你的更新锁定)

| 项 | 锁定值 | 说明 |
|---|---|---|
| 引擎 | **Godot 4** | 迭代快、免费、Claude 对 GDScript 很熟 |
| 语言 | **GDScript** | jam 里比 C# 快,Claude 生成质量高;有强偏好再换 C# |
| 表现 | **3D 模型做 2.5D**:3D 场景 + 角度/正交相机,玩法在地面平面(俯视) | Godot 4 的 GLTF/GLB + AnimationTree 足够;比纯 2D 更像暗黑 |
| 资产 | **全部自购**,开工前买好下好 | 见「开工前」一节——务必提前 |
| 职业 | **亚马逊(弓箭手·远程)** | 手感 = 箭矢命中/穿透/多重刷屏/暴击/风筝,**不是**近战顿挫感 |
| 移动 | **WASD + 鼠标朝向**;AI 用 NavMesh(NavigationRegion3D) | 玩家零寻路风险;随从/敌人走导航网格 |
| 技能资源 | **冷却制,无资源条**(普通射击免费) | 比怒气/秘能少一个系统;想要资源球 juice 再加(加量项) |
| 随从 | **1 个,近战坦克型(圣堂武士式)** | 帮远程亚马逊挡身位,弓手能放心风筝——这是远程职业最关键的配套 |
| NPC | **1 个,仅对话**(无商店/无功能) | 开场给框架文案 + 一句任务引子 |

**开工前(2 天之外,强烈建议):**
1. **买好所有资产包**(主角亚马逊 + 随从 + NPC + 3~4 怪 + Boss + 场景 kit + 音效 + 音乐),下好导入待用——别用 jam 工时下载。
2. **做一次 1~2h 技术验证(tech spike)**:把 1 个带动画的角色导进 Godot,跑通「3D 模型 + 2.5D 相机 + 待机/跑/攻/死动画 + 一次攻击」。这一步验证整个美术管线是否成立,是 2.5D 路线最大的未知。
3. **优先买「同一套骨架/自带动画」的角色包**(最好同一卖家的成套动画角色),避免不同 rig 之间动画重定向(retarget)吞时间。

| **NPC** | 多个 | **保留 1 个,仅对话** | 你指定保留 |

**绝不砍(产品本体):**
1. **远程打击 juice**:命中闪白、箭矢命中粒子、穿透、多重射击扇形刷屏、暴击放大飘字、屏震(强力射击/Boss)、死亡碎裂、冰冻碎裂。
2. **传奇橙光仪式**:橙色光柱 + 专属音效 + 小地图星标 + 换装对比箭头。

伤害公式(沿用极简版):`伤害 = 武器伤害 × 技能倍率% × (1 + 主属性/100) × 暴击`(亚马逊主属性=敏捷)

## 2. 亚马逊(弓箭手)战斗设计

**力量幻想**:站在怪堆外,箭雨清场;靠翻滚拉开距离、靠随从挡住冲上来的怪。

**4 主动技能**

| 技能 | 定位 | 解锁 | 冷却 | 说明 |
|---|---|---|---|---|
| 普通射击 | 基础/产出 | L1 | — | 左键单发直线箭,免费,命中有顿挫感 |
| 多重射击 | 主力 AoE | L2 | 短 CD 或免费 | 扇形多箭,清群核心,刷屏爽点 |
| 翻滚(后撤) | 位移 | L3 | ~3s | 风筝命脉,翻滚带短无敌帧;远程手感的灵魂 |
| 强力射击(穿透箭) | 单体爆发 | L5 | ~5s | 高伤、穿透成排,打 Boss/精英 |

**2 被动**:闪避(X% 几率免一次攻击,亚马逊招牌)+ 精准(暴击率/暴伤)。
**(加量)冰冻箭**:命中减速、暴击则冰冻——控场件,留作 stretch 或替换槽。

> 远程没有近战的 Hit-Stop,顿挫感改靠:命中闪白 + 箭矢命中粒子 + 暴击飘字放大 + 强力射击/Boss 命中的轻屏震 + 穿透时一箭串糖葫芦的连环命中反馈。

**配套传奇 4~5 件(描述性命名,弓系)**

| 物品 | 部位 | 特效 | 服务 build | 标记 |
|---|---|---|---|---|
| 裂空之弓 | 弓 | 多重射击额外 +2 箭 | 多重流核心(武器级、单件成立) | **首橙白名单** |
| 鹰眼之戒 | 戒指 | 多重射击伤害 +150% | 多重流第二件 | — |
| 贯星长弓 | 弓 | 强力射击无限穿透 + 伤害 +200% | 穿透/单体流核心 | 白名单 |
| 疾影之靴 | 靴 | 翻滚冷却减半,翻滚后 3 秒射速 +30% | 操作手感件(一件改变玩法) | — |
| (stretch)霜息箭袋 | 腰/箭袋 | 普通射击减速,暴击则冰冻 + 被冻目标受伤 +25% | 控场件 | — |

**怪物(逼出风筝节奏的组合)**:缓行走尸(慢填充)/ 冲锋疯犬(快,逼你跑)/ 骷髅弓手(远程,逼你走位)/ 自爆肿胀者(逼你走位)。精英词缀:熔火(+ stretch:冰冻/加速)。Boss:屠夫(冲锋 + 横扫 + 烧地板),全部可被「翻滚 + 风筝 + 随从挡」破解。

## 3. 重切后的体验流程(单次 ~40 分钟)

| 时间 | 内容 | 新东西节拍 |
|---|---|---|
| 0–2 | 开场字卡 → 出生城内,**与 NPC 对话拿任务引子** → 移动/射击教学(1 Toast) | 主技能 + 首批击杀 |
| 2–8 | 战斗区 1:杂兵→小怪潮;升 2 级 → 多重射击;**随从入队(身位墙)**;首批蓝/黄装;首精英(1 词缀) | 多重射击、随从、首精英 |
| 8–12 | **首件传奇(脚本保底 ~10 分钟)** → 橙光 → 换上明显变强 | **首传奇** |
| 12–20 | 战斗区 2:新怪种(冲锋疯犬/骷髅弓手),密度更高;升 3~4 级 → 翻滚 + 被动;第 2 传奇窗口 | 翻滚、被动、第 2 橙 |
| 20–26 | 战斗区 3:密度峰值 + 精英组;升 5 级 → 强力射击 | 强力射击、割草高潮 |
| 26–32 | Boss 房 → **屠夫战**(翻滚躲冲锋、走位躲地板火、随从扛伤);击杀必掉橙 | Boss + 必掉橙 |
| 32–34 | 通关页 → **解锁梦魇** | 梦魇入口 |
| 34–45 | 梦魇 1 层(同图怪物 ×1.6 血/×1.3 伤,UI 明示掉率 +50%)→ 强化屠夫 → 终局橙雨 + 结算页 | 梦魇倍率、终局橙雨 |

可伸缩:删一个战斗区→30 分钟;加梦魇 2~3 层→55~60 分钟。

**要做出来的内容清单:**
- 玩家:亚马逊,4 主动(普通/多重/翻滚/强力)+ 2 被动(闪避/精准)
- 随从:1 个近战坦克(跟随 + 自动攻击 + 周期小治疗/嘲讽)
- NPC:1 个仅对话(几句台词 + 任务引子)
- 怪物:3~4 种 + 精英词缀 1~2 + 屠夫
- 传奇:4~5 件(见上)
- 关卡:战斗区 1/2/3 + Boss 房(+ 梦魇复用)
- UI:HUD、技能栏、背包+装备、对比 Tooltip、对话框、开场字卡、通关/结算页、梦魇横幅
- 系统:伤害结算/掉落生成/智能掉落+保底/背包装备/升级解锁/梦魇倍率/对话/检查点复活

## 4. 团队与四域分工

| 域 | 谁 | 主责 |
|---|---|---|
| **战斗(程序)** | 程序① | 玩家弓箭战斗、4 技能、敌人 AI、屠夫 Boss、**随从 AI**、战斗 juice 实现 |
| **系统(程序)** | 程序② | 掉落/物品/词缀、背包/装备、升级、梦魇倍率、数据表加载、**对话系统**、UI 逻辑(背包/Tooltip/HUD)、检查点存档、集成出包 |
| **关卡(策划/美术·偏设计)** | 设计者 | 关卡 blockout、刷怪/遭遇/节奏设计、全局数值与平衡、**随从与 NPC 的行为参数+对话文案**、把角色资产摆进场景、Playtest 主持、数据表(数值) |
| **角色(策划/美术·偏美术)** | 美术者 | 主角/随从/NPC/怪物/Boss 的 **3D 模型+动画接入(AnimationTree)**、VFX 粒子(箭命中/死亡/光柱/升级)、UI 视觉、音效音乐、最终 juice 打磨 |

> 直接对上你的「按战斗/系统/关卡/角色分工」。其中:随从 AI 在「战斗」(本质是 AI),随从/NPC 的美术在「角色」,行为参数与对话内容在「关卡」,对话框 UI 逻辑在「系统」——三方在随从+NPC 上有协作点,Hour 0 对齐好接口即可。

工时假设:**2 天 × 约 12h = 约 24 人时/人 ≈ 96 人时**。下面是净开发估时,真正的安全阀是第 6 节砍量梯子。**「角色」域因 3D 导入/动画最吃时间,plate 最紧,优先受砍量梯子保护。**

## 6. 砍量梯子 & 加量梯子

**落后时按顺序砍:**
1. 梦魇从「重打 + 强化 Boss」简化为「最后一区怪物 ×N + 奖励横幅」,甚至先只做掉率加成。
2. 随从治疗/嘲讽砍掉,只保留「跟随 + 自动攻击」(纯身位墙)。
3. 屠夫烧地板砍掉,只留「半血后攻速变快 + 一次预警 AOE」。
4. 战斗区 3→2。
5. 技能 4→3(保留 普通/多重/翻滚)。
6. 精英词缀 2→1(熔火)。
7. 传奇 5→3(首橙白名单件 + 2 成型件)。
8. 对比 Tooltip 降级为「只显示新旧 DPS 两个数字」。
9. NPC 对话从多句压成一句字幕。

**领先时按顺序加:**
1. 梦魇加到 2~3 层(纯调倍率)。
2. 加冰冻箭(第 4/5 技能)+ 霜息箭袋,验证控场流。
3. 加 1 条符文给多重射击,验证「同技能两种玩法」。
4. 加第 5~7 件传奇。
5. 加 1~2 种怪 + 第 2 条精英词缀。
6. **第二职业(野蛮人/巫师)**——头号大加量项,约等于半个新游戏,建议留到 2 天之后。

## 8. Claude 在全流程中的主导作用(你指定的重点)

**总原则:Claude 主导「生成与决策推演」,人类主导「在 Godot 里运行、买/导资产、手感与平衡的肉眼迭代、最终集成拍板」。** 也就是说,大部分代码、数据、文案、设计推演由 Claude 产出,人是「集成者 + 策展人 + 调味师 + 采购」。

### 8.1 按四域:Claude 主导什么

| 域 | Claude 主导产出(可直接交付的) | 人类必须做的 |
|---|---|---|
| 战斗 | 玩家控制/箭矢/4 技能/敌人 AI/随从 AI/屠夫攻击状态机 的完整 GDScript;攻击手感参数初值;命中判定/无敌帧/冷却逻辑 | 在 Godot 运行、接动画事件、调手感数值到「爽」 |
| 系统 | 掉落生成器/物品词缀/背包/装备/升级/梦魇倍率/数据表加载/对话框/存档 的完整脚本 + 数据表(.tres/CSV)初版 | 接 UI 节点、验证、集成 |
| 关卡 | 关卡节奏方案、刷怪表、遭遇组合、TTK 与平衡推演(给定公式直接算)、NPC 对话文案、任务引子文案、教学 Toast 文案 | 在编辑器摆房间/烘焙导航、跑测、按手感微调数值 |
| 角色 | 技术美术:GLTF 导入设置建议、AnimationTree 状态机脚本、VFX 粒子(GPUParticles3D)与 Shader 代码、UI 布局代码、音频触发逻辑;**资产采购清单与「买什么最省事」的建议** | 实际采购、导入模型/动画、摆 VFX、定视觉/听觉 |

### 8.2 按阶段:Claude 贯穿主导

- **Hour 0**:让 Claude 产 Godot 项目目录骨架、数据表 schema、命名规范、自动加载(autoload)结构,直接定下工程地基。
- **Day 1(批量造地基)**:把本文档的「内容清单 + 伤害公式 + 数据表格式」喂给 Claude,让它**一次产一个完整可运行脚本 + 挂载说明**(玩家控制、箭矢、掉落、背包、升级…),人快速集成成灰盒。系统域大部分代码这天可由 Claude 主导生成。
- **Day 2(造内容+调味)**:Claude 产屠夫攻击状态机、梦魇倍率逻辑、对比 Tooltip、传奇特效、VFX/Shader 片段;同时做平衡推演(给它怪物 HP / 玩家 DPS,让它算各区 TTK 并提调参建议)、写全部文案。
- **收尾(debug 主导)**:报错就把**完整报错栈 + 相关脚本**贴回 Claude,让它定位并给修复版;让它写 readme、提交说明、操作指引。

### 8.3 高效协作模式(给两位程序)

1. **每域开一个长会话**保持上下文,别让 Claude 在不知道你工程结构时瞎猜。
2. **先要接口约定再各自实现**:让 Claude 先输出「各系统对外的函数/信号(signal)契约」,两位程序据此并行,减少集成冲突。
3. **要「整段可编译 + 怎么挂」**,不要碎片:每次明确「Godot 4 / GDScript / 这个脚本挂在哪个节点 / 依赖哪些信号」。
4. **报错贴全栈**,并说明 Godot 版本与节点树。
5. **数据表让 Claude 批量产**:词缀池、传奇表、怪物数值、XP 曲线,直接让它生成 .tres/CSV,人只审。

> **Git 协作约定(全员)**
> - 仓库已加 `.gitattributes`,把 Godot 文本资源(`.gd`/`.import`/`.tres`/`.tscn`/`.godot`/`.uid` 等)统一锁为 **LF** 行尾,二进制资源(`.fbx`/`.png`/`.ttf` 等)标 `binary`。这是为根治此前 500+ 个 `.import` 文件因 `core.autocrlf` 反复在 LF/CRLF 间抖动产生的噪音 diff。
> - **首次 pull 到此修复后,本地若有 CRLF 旧副本会出现一次性"重规范化"改动**,执行下面两行清掉即可(无内容变化,放心丢弃):
>   ```bash
>   git add --renormalize .
>   git checkout -- .
>   ```
> - 之后正常提交即可,Godot 重新导入资源**不会再产生**满屏 `.import` 行尾 diff。
> - `数值表/*.csv` 已设 `importer="keep"`(不当翻译导入);**新增 CSV 时记得把其 `.csv.import` 也设为 keep**,否则中文表头会再次触发非法翻译路径报错。

### 8.4 Claude 不该主导(必须人来)的边界

- 在引擎里**实际运行验证**(Claude 看不到运行结果)。
- **手感与平衡的最终肉眼判断**(「爽不爽、难不难」是主观迭代)。
- **资产采购与版权确认**。
- **最终集成与取舍拍板**(砍哪个、留哪个,人对着 Playtest 决定)。

> 一句话:这个 jam 里 Claude 是「主力生产线 + 设计副驾 + debug 搭子」,把人从写样板代码里解放出来,专注在「跑、调、买、拍板」这四件机器替不了的事上。
