### 3.4 ItemGenerator API(任务2,掉落系统/调试用)
```gdscript
var gen := ItemGenerator.new(DataTables)           # 传入 DataTables autoload, 可选第2参 seed
```

```gdscript
ProgressionManager.add_xp(amount)        # 加 XP,跨级自动循环升级
```

```gdscript
# DropSystem —— 纯逻辑掉落判定(RefCounted, 可 headless 测), 一般由 LootManager 调度
```

### 3.7 Inventory 背包+装备 API(任务4,角色D/HUD/战斗①对接)
```gdscript
# 背包(40 格)
Inventory.add_item(item) -> bool          # 满包(40)返回 false,物品留地面
Inventory.bag_count() -> int
Inventory.is_full() -> bool
Inventory.get_bag_items() -> Array[ItemInstance]   # 只读快照(UI 渲染用)

# 装备(13 槽, slot = EquipSlots.Slot 0~12)
Inventory.equip(slot, item) -> bool       # item 须在背包;旧装备回背包(满包则失败保持原状)
Inventory.unequip(slot) -> ItemInstance   # 回背包(满包返回 null);空槽/非法槽返回 null
Inventory.get_equipped(slot) -> ItemInstance       # 空返回 null
Inventory.quick_equip(item) -> int        # 按 item.slot 自动选槽;戒指自动路由 RING_1→RING_2;返回落位 slot,失败 -1

# 属性聚合(基础主属性 + 已装备词缀)
Inventory.get_total_stats() -> Dictionary # { AffixDef.StatKind(int): value(float) }
```
- **属性生效**:`get_total_stats` 取 `ProgressionManager.agility/vitality` 作基础,叠加所有已装备件的 `aggregate_stats()` 词缀加成。任一装备/卸下都发 `stats_changed(total_stats)` → HUD/角色面板/战斗① 刷新。
- **换装语义**:换装时旧装备回填到新装备腾出的格子,**净占用不变**;直接装备地面物(不在背包)时旧装备需新增一格,满包则装备失败。
- **戒指双槽**:RING_1/RING_2 独立;`quick_equip` 自动选空槽,两戒满则替换 RING_1。
- **UI**:`scenes/ui/inventory_panel.tscn`(程序化 CanvasLayer,layer=110 盖 HUD)——`B` 键(`toggle_inventory` 动作)开关,左 40 格网格(点击=quick_equip)、右 13 槽(点击=卸下)、下属性面板(监听 `stats_changed` 实时刷新)。
- headless 实测:`tools/verify_inventory_equip.gd`(40 格溢出 / 装备-卸下信号 / 基础+词缀聚合 / 换装净占用不变 / 戒指双槽 quick_equip,全部 `VERIFY OK`)。

### 6.4 Bug 修复:怪物死亡 `det == 0` 物理崩溃刷屏(2026-06-13)

**现象**:游戏运行中控制台每物理帧刷 `det == 0` 矩阵求逆错误(Godot 内部 Transform3D/Basis 反演失败)。

**根因**:怪物死亡走 `enemy_base.gd::_die()` → 用 tween 把本体 scale 缩到 `Vector3.ZERO`(DEATH_DURATION 后 queue_free)。但挂在同一 body 上的独立 `KnockbackComponent._physics_process` 仍在对这个**已退化(零缩放→基矩阵行列式为 0)**的 body 调 `move_and_slide()`,内部对退化基求逆即 `det == 0`,每帧刷屏。死亡与击退两条物理路径未互斥。

**修复**:
- **主修复** `scripts/entities/enemy_base.gd::_die()`:进入 `State.DEATH`、清零 velocity 后,若 `knockback_comp` 存在且有 `cancel()` 即调用,立即停止击退位移,scale 归零前击退已不再驱动物理。
- **防御兜底** `scripts/components/knockback_component.gd::_physics_process`:`_body == null` 守卫后追加 `_body.is_queued_for_deletion()` 判断,待销毁则 `cancel()` 并 `return`,杜绝对将销毁/退化 body 调 `move_and_slide()`。

**回归验证** `tools/verify_knockback_death.gd`(`VERIFY OK`,EXIT=0):
- 判定①:`apply` 后击退正常启动(is_active==true,初速/剩余时间已设)。
- 判定②(修复点):本体标记 `queue_free()` + scale→0 后,`_physics_process` 自动 `cancel()`,is_active 变 false 且后续帧保持停止,不再触发 `move_and_slide`。
- 注:SceneTree-script 测试 harness 中 `add_child` 不同步触发 `_ready`/无物理空间,故判定①以状态断言代替驱动真实 `move_and_slide`,避免空间缺失伪报。
