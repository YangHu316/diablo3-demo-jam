## 三、对外接口契约(队友重点看这节)

### 3.1 DataTables 查询 API(战斗①对接)
```gdscript
DataTables.get_all_affixes() -> Array[AffixDef]
DataTables.get_affix(id) -> AffixDef
DataTables.get_affixes_for_slot(slot_name) -> Array[AffixDef]   # roll 词缀用
DataTables.get_legendary(id) -> LegendaryDef
DataTables.get_first_orange_whitelist() -> Array[LegendaryDef]  # 首橙保底
DataTables.get_monster_stats(id, level) -> { health, attack, xp }
#   怪物 id: trash / elite_blue / champion_yellow / skeleton_guard(骸骨卫士) / butcher(屠夫)
DataTables.get_xp_to_next(level) -> int     # -1 = 已满级
DataTables.get_tier_for_level(level) -> int
DataTables.xp_curve / DataTables.tier_table  # 直接访问曲线对象
```

### 3.2 信号契约
```gdscript
# ProgressionManager —— 战斗① 接 level_up 解锁技能
signal xp_gained(current_xp: int, xp_to_next: int, level: int)
signal level_up(new_level: int, unlocked: Array)
signal stats_changed(stats: Dictionary)   # 升级后主属性变化(HUD/战斗①刷新)
#   unlocked 示例: ["slot_2", "skill_multishot"]  (来自 xp_curve.unlocks)

# Inventory —— 角色D 接 loot_dropped 挂光柱/音效
signal loot_dropped(quality: int, world_pos: Vector3)   # quality=ItemInstance.Quality
signal item_picked_up(item: ItemInstance)
signal item_equipped(slot: int, item: ItemInstance)     # slot=EquipSlots.Slot
signal item_unequipped(slot: int, item: ItemInstance)
signal stats_changed(total_stats: Dictionary)           # HUD/面板刷新
```

### 3.4 ItemGenerator API(任务2,掉落系统/调试用)
```gdscript
var gen := ItemGenerator.new(DataTables)           # 传入 DataTables autoload, 可选第2参 seed
gen.generate(slot, item_level)                     # 品质按权重(蓝85/黄14.5/橙0.5)随机
gen.generate(slot, item_level, ItemInstance.Quality.RARE)  # 指定品质
gen.generate_legendary(&"windforce_boots", 8)      # 指定传奇
gen.roll_quality() -> int                          # 单独抽品质
```
- 蓝 1~2 词缀 / 黄 3~4 / 橙 4+橙字特效(不占位);按部位过滤词缀;按等级定 Tier roll 数值。
- 返回 `ItemInstance`,其 `aggregate_stats()` 给 Inventory 算面板。

### 3.5 ProgressionManager API(任务5,战斗①/HUD 对接)
```gdscript
ProgressionManager.add_xp(amount)        # 加 XP,跨级自动循环升级
ProgressionManager.get_stats() -> { level, current_xp, xp_to_next, agility, vitality, max_hp }
ProgressionManager.is_max_level() -> bool
ProgressionManager.level / current_xp / agility / vitality / max_hp   # 直接读字段
```
- **击杀自动给 XP**:`_ready` 里连 `CombatManager.enemy_killed`。怪物实例若挂 `monster_id`/`monster_level` 元数据则据此查表;**未挂元数据时兜底按 `trash`@玩家当前等级**,战斗①接入前即可跑通。
- 跨级:单次大 XP 连升多级,逐级发 `level_up`。满级(8)后 XP 不再溢出/不越级,经验条置满。
- 升级即按 `xp_curve` 刷新敏捷/体能/生命并发 `stats_changed`。
- headless 实测:`tools/verify_progression.gd`(L1→800XP→L3 余20、属性成长、解锁 payload、满级封顶、缺省兜底,全部 `VERIFY OK`)。

### 3.6 掉落系统 DropSystem / LootManager(任务3,战斗①/关卡C 对接)
```gdscript
# DropSystem —— 纯逻辑掉落判定(RefCounted, 可 headless 测), 一般由 LootManager 调度
var ds := DropSystem.new(DataTables, item_generator, seed)
ds.roll_drop(DropSystem.Source.TRASH, player_level) -> Array[ItemInstance]  # 空=不掉
ds.register_elite_kill()        # 杀完一组精英喂软保底(每组 +2% 橙)
ds.tick(delta)                  # 每帧推进保底计时(硬保底 8 分钟首橙)
ds.set 字段: nightmare_tier(0/1/2 梦魇层叠橙权重)
ds.get_legendary_count()        # 本局已掉传奇数(结算 n/5 钩子)

# LootManager(autoload)—— 击杀→掉落→地面 spawn 全链路, 战斗①只需挂元数据
LootManager.notify_elite_group_killed()   # 战斗① 杀完一组精英时调一次
LootManager.set_nightmare_tier(tier)      # 关卡C 进梦魇层时设置
```
- **掉落源**:`TRASH / ELITE_BLUE / CHAMPION_YELLOW / BUTCHER / CHEST_COMMON / CHEST_FANCY`,掉落率+件数+蓝黄橙权重严格对齐策划 §6.1。
- **战斗① 对接**:怪物实例挂 `drop_source`(int=DropSystem.Source)+ `monster_level`(int)元数据,LootManager 接 `CombatManager.enemy_killed` 自动在怪物位置 spawn 地面掉落物。**未挂时兜底 TRASH@玩家等级,不阻塞联调。**
- **保底**:首件传奇必∈白名单(女妖弓/冰霜箭袋/疾风靴);前 4 件传奇查重不重复;硬保底 8 分钟无橙则下个保底载体强制首橙。
- **拾取**:`loot_drop.tscn` 走近(范围 Area3D)+ 按 `interact`(F 键)拾取 → `Inventory.add_item` → 满包(40)留在地上(DUI-05 手动拾取口径)。
- headless 实测:`tools/verify_drop_system.gd`(TRASH 18.4% / 精英 100% 且 59-38-3 / 首橙白名单 200局命中 / 硬保底首橙 / 前4件查重 / 入包满包封顶,全部 `VERIFY OK`)。
