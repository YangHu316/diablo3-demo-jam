## 二、技术:rect→栅格 navmesh 编译器 + 自动墙(为什么这么做)

大关卡(18 房间/走廊)若手编 navmesh 需 200+ 共享顶点多边形,极易断。本关用**程序化编译器**(`level_02_depths.gd`):

1. 关卡 = 一组**可走矩形 `WALK`**(房间+走廊,相邻处重叠)。
2. 把所有矩形的 X/Z 边界并成一张**栅格**;每个落在任意矩形内的栅格单元 = 一个 navmesh 四边形,**顶点按栅格坐标去重共享** → 相邻单元天然共边连通(连通性由构造保证,实测 224 面全连通)。
3. **墙体由同一栅格自动生成**:可走单元与「不可走邻格」的边界 = 一段墙。开口恰在矩形相接处。
4. 全部几何 `collision_layer=4`(玩家 mask=4 / 敌人 mask=5 与之碰撞)。

> 改关卡 = 改 `WALK` 数组(再跑 `verify_level02.gd`)。`SCALE` 常量统一缩放(当前 1.5,放大避免局促、拉长纵深)。
> 备注:运行时烘焙(bake)在 headless 下 map 同步不稳;手编共享顶点 navmesh 实测稳定,故采用此法。

## 三、核心路径、区域与节奏

```
西门(入口) → 西廊[首遇] → 中央枢纽 ┬─北→ 长北廊[火光·伏击] → {上中室; 北门(目标·光柱牵引)[守卫]}
								   ├─西南→ 宝藏死胡同
								   ├─东→ 东廊[过渡] → 齿轮室[机关] → 右环(绕中心实体)[尸群] → Boss入口廊 → Boss厅(中心柱)[尸潮高潮·出口]
								   └─南→ 枢纽南廊 → 南长廊[下环] ──→ 汇入 Boss 入口(与东路构成大环路)
```

- **入口/出口**:入口=西门;出口=Boss 厅深处(击杀高潮后出关)。**目标**=北门(远北,纵深主轴尽头,光柱远端可见)。
- **环路**(原图招牌):① 右环 RL 绕中心实体回游;② 下环 S1→S2 与东路两条到 Boss(实测 东路 130 / 下环 116,长度相近,互为替代——"绕路不亏"的好环)。
- **区域类型**:战斗(枢纽/东廊/齿轮室/右环/南长廊/北廊/北门/Boss入口/Boss厅);纵深廊(N1 北廊串火光);资源/支线(宝藏死胡同、北门目标);休整靠节奏谷(廊段)与篝火光点。
